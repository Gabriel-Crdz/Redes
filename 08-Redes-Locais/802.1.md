# Redes Locais

## Padrão IEEE 802.1
Esse grupo de trabalho especificou vários serviços de controle da
circulação dos quadros dentro de uma rede local.

Os principais padrões especificados são:
- **802.1D** - MAC Bridges (Spanning Tree)
- **802.1p** - Priorização de pacotes
- **802.1Q** - LANs Virtuais (VLAN)
- **802.1X** - Controle de acesso à rede

---

## 802.1D - Spanning Tree Protocol (STP)

Foi criado para evitar problemas em redes com múltiplos switches conectados.

Em uma **rede ideal**, os switches formam uma estrutura em árvore, garantindo apenas um caminho entre dois dispositivos. Porém, quando existem múltiplas conexões entre switches, surgem **laços (loops)**, permitindo mais de um caminho para os dados.

O problema desses laços aparece principalmente **em transmissões broadcast:** os quadros podem se duplicar indefinidamente, **circulando pela rede em um ciclo infinito**, o que causa sobrecarga e pode derrubar toda a rede.

![rede com loop](/08-Redes-Locais/img/figura2.png)

Para resolver isso, o STP:
* Monitora a rede em busca de loops
* Desativa automaticamente (por software) alguns links redundantes
* Mantém apenas um caminho ativo, formando uma estrutura em árvore

Se algum link ativo falhar, o STP reativa um dos links desabilitados, garantindo redundância e continuidade da rede.

![rede sem loop](/08-Redes-Locais/img/figura3.png)

## 802.1Q - Virtual Lan (VLAN)

Em **redes tradicionais**, todos os **computadores se comunicam entre si**, o que pode gerar:

* **Problemas de desempenho:** excesso de tráfego broadcast e propagação de malwares
* **Problemas de segurança:** maior exposição a invasões e vazamento de dados

A **VLAN** resolve isso **ao segmentar a rede**, fazendo com que cada grupo de dispositivos só se comunique com outros da mesma VLAN, como se fossem redes separadas, mesmo **usando a mesma infraestrutura física**.

Em switches simples, essa divisão pode ser feita por portas. Porém, em redes maiores com vários switches, é necessário **identificar a qual VLAN cada pacote pertence**. Para isso, o padrão IEEE 802.1Q adiciona uma tag **(etiqueta)** ao quadro Ethernet.

![localização da tag](/08-Redes-Locais/img/figura4.png)

Essa tag possui 4 bytes e inclui:

TPID: identifica o uso do padrão 802.1Q

TCI, que contém:
* **Prioridade** (qualidade de serviço)
* **DEI:** Indicador de descarte em congestionamento
* **VID (VLAN ID):** identifica a VLAN (até 4096 possíveis)

![tag](/08-Redes-Locais/img/figura5.png)

## 802.1p - Priorização de tráfego

A **qualidade de serviço (QoS**) prestada por uma rede pode variar ao longo do tempo, em
função da demanda de tráfego a cada momento.

Como diferentes aplicações têm **necessidades distintas** (por exemplo, chamadas de voz e vídeo exigem baixa latência, enquanto e-mails toleram atrasos), a **rede pode ficar sobrecarregada** e precisar decidir quais dados devem ser transmitidos primeiro.

O IEEE 802.1p resolve isso ao:

* Definir **níveis de prioridade** para os pacotes de dados
* Utilizar **3 bits do campo de prioridade** dentro da tag do IEEE 802.1Q
* Permitir **até 8 níveis diferentes de prioridade**

Assim, aplicações mais sensíveis ao atraso, como VoIP e videoconferência, recebem prioridade maior em relação a tráfegos menos urgentes, como transferência de arquivos.

## 802.1X - Controle de Acesso

Define um mecanismo de **controle de acesso à rede**, garantindo que apenas usuários autorizados possam se conectar.

Em redes comuns, qualquer dispositivo conectado fisicamente pode acessar a rede. O 802.1X resolve isso ao exigir autenticação antes de liberar o acesso, atuando na camada de enlace.

#### O funcionamento básico:
1. Quando um dispositivo se conecta, a porta do switch fica bloqueada (não autorizada)
2. Apenas tráfego de autenticação é permitido
3. O switch solicita credenciais usando o protocolo EAP
4. O cliente envia suas credenciais
5. Um servidor de autenticação, como o RADIUS, valida essas informações
6. Se estiverem corretas, a porta é liberada e o acesso à rede é permitido
